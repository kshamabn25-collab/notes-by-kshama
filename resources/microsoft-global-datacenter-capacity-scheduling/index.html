Global Data Center Capacity Scheduling

Scheduling Program Manager case study pack — integrated master schedule, CPM analysis, and portfolio governance (readable pages).

What this pack contains

This is a structured set of scheduling and program governance artifacts for a global data center capacity delivery portfolio. The aim is to show how I build an Integrated Master Schedule (IMS), surface cross-segment dependencies, run critical path / sensitivity analysis, and communicate schedule health clearly enough to drive decisions.

Work Cloud Infrastructure Portfolio Delivery Scheduling / CPM EPPM / P6 Style

At a glance

  • Portfolio: Global Data Center Capacity Delivery (multi-region)
  • Timeline: Jan 2025 – Dec 2026
  • Scope: 5 regional programs, end-to-end milestones (civil → power → network → commissioning → handover)
  • Schedule approach: Integrated Master Schedule (IMS) with CPM logic + near-critical monitoring
  • Risk approach: Sensitivity scenarios + mitigation/recovery sequencing
  • Outputs: Executive dashboard, dependency register, change impact analysis, status reporting
